# lit-hunt · 找文献，不再抓瞎

一个**通用型**论文资料查找工具——给所有学科的研究者用，不限于某个领域。

## 它要解决什么

做研究的时候，下面这四件事最容易让人抓瞎：

1. **不知道关键词怎么写**——脑子里的问题，翻译成数据库能搜的词，差着十万八千里
2. **找到一堆文献不知道怎么筛**——搜出来 200 篇，哪 10 篇值得看？
3. **不确定引用格式写对没**——APA7 还好，碰到 MLA / Chicago / Harvard 心里没底
4. **论文大纲不知道怎么下手**——选题有了，但不知道从哪段写起

这个工具就是把这四件事打包起来。

## v0.2.1：Lovable 风格的可视化编辑层

点页面右上角 **Edit** 进入编辑模式。鼠标移到任何元素上会有虚线高亮，点击选中（实线高亮）→ 右侧浮窗显示元素信息 + 12 个快捷按钮 + 一个大白话输入框。

**快捷按钮**：`A+` / `A−` / `深` / `茶` / `红` / `B`（加粗）/ `I`（斜体）/ `居` / `影`（加阴影）/ `隐` / `显` / `删`

**自然语言输入**（例子）：
- "大一点" / "小一点" → 字号 ±20%
- "换红色" / "换茶色" / "深一点" → 颜色
- "居中" / "加粗" / "加阴影" → 对应样式
- "隐藏" / "删除" → 显隐 / 删除
- 没听懂会提示"试试这些词"

**提交**："📋 发给 Coder" 按钮把所有改动整理成结构化清单（按元素分组）复制到剪贴板，贴到对话里，我（agent）就把这些改到源文件、commit、push。

**重置**：↺ 按钮刷新页面，未提交的改动会丢（会先确认）。

### 工作流

1. 打开本地服务器：`python3 -m http.server 8000`
2. 进 `http://localhost:8000`
3. 右上 **Edit** → 选中元素 → 改 → **发给 Coder** → 复制
4. 回来把清单贴给我
5. 我改源文件、commit、push
6. 你刷新页面看到正式版

### 文件

- `edit.css` — 编辑器样式
- `edit.js` — 编辑器逻辑（独立于主 `script.js`）
